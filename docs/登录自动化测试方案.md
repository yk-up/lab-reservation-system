# 登录自动化测试方案（实验室预约系统）

## 1. 需要测试的功能点分析

围绕当前实现（`frontend/src/views/Login.vue`、`/api/auth/login`、路由守卫）可拆分为以下测试点：

- 登录表单校验：用户名必填、密码必填、回车提交行为。
- 登录接口交互：请求参数正确、错误响应可见、成功响应可消费。
- 登录成功后的状态落库：`token` 和 `userInfo` 是否写入本地存储。
- 登录成功后的路由分流：
  - 普通用户跳转 `/labs`
  - 管理员跳转 `/admin/dashboard`
- 登录失败处理：错误提示可见且不离开登录页。
- 鉴权联动：后续访问受保护页面时可复用登录态。

## 2. 测试用例清单

| 用例ID | 用例名称 | 前置条件 | 步骤 | 预期结果 |
|---|---|---|---|---|
| LOGIN-001 | 普通用户登录成功 | 可访问登录页 | 输入 `student001/123456` 点击登录 | 跳转 `/labs`，出现欢迎提示 |
| LOGIN-002 | 管理员登录成功 | 可访问登录页 | 输入 `admin/123456` 点击登录 | 跳转 `/admin/dashboard` |
| LOGIN-003 | 错误密码登录失败 | 可访问登录页 | 输入正确用户名+错误密码 | 保持在 `/login`，出现“用户名或密码错误” |
| LOGIN-004 | 用户名为空提交 | 可访问登录页 | 用户名空、输入密码后提交 | 表单提示“请输入用户名” |
| LOGIN-005 | 密码为空提交 | 可访问登录页 | 输入用户名、密码空后提交 | 表单提示“请输入密码” |

> 本次已自动化实现前 3 条（满足“至少 3 个测试用例”要求），第 4~5 条建议作为下一轮补充。

## 3. Page Object 封装设计

- `frontend/e2e/pages/login.page.js`
  - 元素：用户名输入框、密码输入框、登录按钮
  - 行为：打开登录页、执行登录、Mock 成功登录响应、Mock 失败登录响应
- 价值：
  - 统一定位器，降低页面结构调整带来的维护成本
  - 测试用例仅描述业务行为，增强可读性

## 4. 已实现自动化用例

- `frontend/e2e/tests/login.spec.js`
  - `正常用户登录成功后跳转到实验室列表`
  - `管理员登录成功后跳转到管理端看板`
  - `登录失败时显示后端错误提示并停留登录页`

## 5. 执行方式

在 `frontend` 目录执行：

1. 安装依赖：`npm install`
2. 安装 Playwright 浏览器：`npx playwright install`
3. 运行测试：`npm run test:e2e`
